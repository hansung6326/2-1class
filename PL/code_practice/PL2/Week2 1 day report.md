### 프로그래밍 패러다임

* 주어진 유형의 문제를 다루는데 효과적인 일관된 방법들의 집합

#### 명령형/절차형(imperative/procedural) 패러다임

* 기억장소를 나타내는 변수 사용
* 명령의 순차 실행
* 변수 값 변경을 위해 배정문 이용 -> 메모리 조작
* 명령형/절차형 프로그래밍 언어

#### 함수형(functional) 패러다임

* 프로그램의 동작을 추상적으로 정확하게 기술
* 함수형 언어: 연속적인 함수 호출의 형식

#### 객체지향(object-oriented) 패러다임

* 프로그램의 상태를 객체 안에 저장/보호
* 정의된 연산만을 통해서 접근

* Java
