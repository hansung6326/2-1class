### AI 활용 가이드

* AI는 코딩 기계가 아닌 '지능형 튜터'

* 지능형 튜터링
  * 개인별 학습 속도에 맞춘 1:1 맞춤형 질의응답
  * 수업 중 발생한 학습 공백 보완

* 개념 시각화 및 구체화
  * 추상적 프로그래밍 언어 이론(메모리 모델, 스코프 등)의 직관적 이해
  * 구체적 예시 및 코드 변환을 통한 딥러닝

* 비판적 사고 및 검증
  * AI생성 결과물 속 잠재적 오류 탐지
  * 이론적 근거를 바탕으로 코드 동작 원리 파악

* 허용 및 권장 범위
  * 이론 학습 도우미
  * 코드 리팩토링 및 분석
  * 디버깅 가이드
  * 비교 학습

* 금지 및 제한 사항
  * 무단 복사 및 붙여넣기
  * 학습 목표 위배

* AI의 답변을 의심하라: 3단계 코드 검증 프로세스
  * Generate(생성)
  * Analyze(분석)
  * Cross-check(교차 검증)

* 성공적인 AI 활용을 위한 3대 황금률
  1. 시각화 도구
     * 생각의 외주화가 아닌, 어려운 개념의 시각화 도구로 활용할 것
  2. 주석 작성
     * 단순 복붙은 거절, 모든 코드에 이론적 근거를 바탕으로 주석 달기
  3. 직접 검즘
     * AI의 답을 맹신하지 말고, 배운 지식과 도구로 직접 교차 검증하기

### AI 활용 윤리와 학문적 정직성

* 올바른 AI 활용을 위한 3대 핵심 원칙
  1. 정보의 투명성
     * AI도구 활용 여부 명시 필수
     * 결과물 도출에 사용된 프롬프트 내역 기록
     * AI 답변을 바탕으로 수행한 수정 및 보완 내역 투명화
  2. 저작권 및 책임
     * AI 생성물의 라이선스 및 저작권 위반 여부 확인 의무
     * 생성된 코드 및 결과물의 논리적 결함 직접 검증
     * 결함으로 인해 발생한 모든 결과에 대한 최종 책임 인지
  3. 기술적 종속 경계
     * AI가 제시한 답변의 무비판적인 수용 태도 지양
     * 항상 '왜 이렇게 동작하는가?' 질문하는 습관 형성
     * 스스로 논리를 검증하고 자체적인 해답 도출

* AI 실전 활용 가이드(1) : 태도 & 과제
  * 태도
    * 권장: AI를 철저한 검증 대상으로 대하기
    * 금지: AI를 완벽한 정답지로 맹신하기
  * 과제
    * 권장: AI의 설명을 완벽히 내 것으로 소화해 다시 쓰기
    * 금지: AI의 출력 결과물을 그대로 복사해서 제출하기

* AI 실전 활용 가이드(2) : 학습 & 디버깅
  * 학습
    * 권장: 코드의 원리와 구조를 이해하기 위해 질문하기
    * 금지: 오직 최종 결과물만 얻기 위해 질문하기
  * 디버깅
    * 권장: 왜 에러가 났는지 발생 이유 묻기
    * 금지: 에러 메시지 복사 후 무작정 수정해달라고 하기

* 핵심 요약: AI는 '정답 자판기'가 아닌 '성장의 도구'
  * 과정 공유
  * 주도권 확보
  * 본질 집중

### AI워크플로우의 두 갈래: IDE vs Terminal

* ##### IDE형

  * 통합 개발 환경 기반
  * GUI 중심의 시각적 워크플로우
  * GUI 기반 통합 환경
  * Visual Studio Code, Cursor IDE, PyCharm + AI 플러그인
  * 프로젝트 전체 구조 시각화
  * 시각적 디버깅 및 브레이크포인트 제어
  * 코드 자동완성 및 생성
  * 인라인 코드 설명 및 리팩토링 지원
  * IDE형 도구의 실무적 명암(장점)
    * 진입장멱 매우 낮음
    * 코드 컨텍스트 기반의 정밀한 AI 지원
    * 강력한 시각적 디버깅 및 협업 지원
  * IDE형 도구의 실무적 명암(단점)
    * 리소스 점유율 높음, 무거운 환경
    * 대규모 배치 작업에는 비효율적
    * 자동화 및 스크립팅 연동에 불리

* ##### Terminal형

  * CLI 기반
  * 텍스트 명령어 중심의 제어 워크플로우
  * OpenAI CLI, LangChain CLI, Shell GPT
  * 텍스트 기반의 코드 생성 및 즉시 실행
  * 기존 시스템 도구와의 강력한 파이프라인 결합
  * 자동화 스크립트 작성 및 서버 환경 활용 극대화
  * Terminal형 도구의 실무적 명암(장점)
    * 극도로 가볍고 빠른 실행 속도
    * 자동화 및 대규모 배치 처리에 최적화
    * DevOps / MLOps 인프로 환경과 완벽한 친화성
  * Terminal형 도구의 실무적 명암(단점)
    * 높은 학습 곡선
    * IDE 대비 코드 전체 컨텍스트 이해 제한적
    * 시각적 디버깅 도구의 부재

### Cursor AI

* AI 기반 코드 에디터
* VS Code 기반 확장형 개발 환경
* 프로젝트 전체 문맥을 이해하는 LLM 통합

* 단순 자동완성 도구가 아니라 대화형 코드 파트너

* Cursor가 하는 일

  * 코드 생성 및 설명
  * 버그 분석 및 리팩토링 제안
  * 테스트 코드 생성
  * 문서화 자동 생성

* AI가 작성한 코드는 무조건적 '검증 대상'

* 핵심 기능 1: Inline Edit(즉석 코드 생성)

  * Ctrl + K

  * 용도: 코드 창에서 즉석으로 코드 생성 및 수정 요청

* 핵심 기능 2: AL Chat(코드 대화 및 튜터링)

  * Ctrl + L
  * 용도: 코드의 원리 파악 및 질문(1:1 튜터 역할)

* 핵심 기능3: @Symbols (Context Targeting)

  * @Files(특정 파일 내용 참고)
  * @Codebase(프로젝트 전체 구조 및 연관성 파악)
  * @Web(최신 라이브러리/ 기술 인터넷 검색)
  * @Docs(공식 문서 기준 답변 요청)

### Antigravity

* 개발자의 생산성을 위한 차세대 AI 코딩 어시스턴트
* 노가다, 단순 반복, 구문 오류라는 '중력'에서 탈출
* Antigravity 개념
  * AI기반 코드 생성 및 수정 도구
  * 코드 설명, 리팩토링, 테스트 생성 지원
  * 대화형 인터페이스 기반 개발 지원

* 코드를 대신 작성하는 도구가 아닌, 사고를 확장하는 도구
* 정체성: 일을 대신하는 로봇 사원
* 사용자 역할: 감독관(목적지 설정 및 승인)
* 스스로 생각하고, 행동하고, 수정한다
  1. 자율 계획
  2. 도구 활용
  3. 자기 수정
