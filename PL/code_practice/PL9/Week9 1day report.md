#### 선언문

* 선언
  * 컴파일러에게 변수(데이터)의 속성에 관한 정보를 알려줌
* 예: int a[10];
  * 배열의 이름 = a
  * 배열 원소의 데이터 유형 = int
  * 배열의 차원 = 1
  * 배열의 원소 개수 = 10
  * 배열의 index 범위 = 0 ~ 9
* 역할
  * 효율적인 기억 장소 관리
  * 정적 타입 검사 = 컴파일 시간 바인딩
* 암시적 선언
  * FORTRAN: I~N으로 시작하면 정수
  * Perl
    * $로 시작: 스칼라 형(= 하나의 값)
    * @으로 시작: 배열 형(= 여러 개의 값)

  * 정적 바인딩
    * 변수의 명시적 선언
    * 컴파일 시간에 속성 결정
    * 효율적(빠른) 코드
    * 컴파일 언어: C, C++...

#### 배정문

* 변수 값을 변경 = 메모리 조작
* 명령형 언어에서 가장 중요한 명령/연산
* l-value(주소) = r-value(값)
* 특징
  * 모든 변수는 l-value, r-value를 가짐
  * 상수: r-value만 가짐(배정문의 왼편에 올 수 없음)
  * 포인터 변수: int *ip;
  * l-value: ip의 주소
  * r-value: 다른 변수의 주소(= 다른 변수의 l-value)
  * A[i]
    * l-value: A의 i번째 요소의 주소
    * r-value: A의 i번째 요소의 값

* 이명 생성

  * 가시적(변수가 참조될 수 있다)
* 정적 영역 규칙
  * 프로그램 블록의 포함 관계에 따라 변수의 영역 결정
  * 비지역 변수: 변수를 둘러싸고 있는 가장 가까운 블록에서부터 선언되었는지 확인
  * 컴파일 시간에 변수의 영역 결정
  * C, C++, Pascal...
* 동적 영역 규칙
  * 함수 호출 순서에 따라 변수의 영역 결정
  * 비지역 변수: 호출한 함수에서 변수를 찾음
  * 실행 시간에 변수의 영역 결정
  * Python, Javascript, APL, SNOBOL4...
* 수명
  * 메모리 할당 시점 ~ 반환 시점
  * 블록 구조 언어 : scope = lifetime
  * static in, C++

    * 수명: 프로그램 종료 시점

#### 기억장소 바인딩과 변수 종류

* 정적 변수
  * 실행 시간 전에 기억장소가 바인딩 됨(= 기억장소가 결정됨)
  * 프로그램 종료까지 바인딩 유지(= 기억장소가 사라지지 않음)
  * 전역 변수
  * 효율성
  * C, C++: static 키워드로 선언한 변수
  * 단점
    * 낮은 융통성 -> recursion 허용되지 않음
    * 변수들은 기억 장소 공간을 공유할 수 없음
* 스택 동적 변수
  * 선언문이 실행될 때 기억장소가 바인딩 됨
  * 정적 바인딩
  * 실행 시간 스택에 기억장소가 할당됨
  * 재귀 프로그램 가능
  * C, C++: 지역 변수
  * 덜 효율적
    * 기억장소 할당/해제가 실행 시간이 이루어지기 때문에
* 명시적 힙 동적 변수
  * 명시적인 실행 시간 명령어에 의한 기억장소 할당/해제 (=바인딩과 바인딩 해제)
  * 힙이라는 기억장소 영역에서 기억장소가 할당/해제
  * 포인터 변수와 참조 변수만 접근 가능함
  * C: malloc(), free() 함수
  * C++: new, delete 연산자
