# GEMINI.md - 프로젝트 가이드라인

이 파일은 Gemini CLI 에이전트가 이 프로젝트의 구조와 목적을 이해하고, 사용자의 학습을 효과적으로 돕기 위한 지침을 담고 있습니다.

## 1. 프로젝트 개요 (Project Overview)
이 프로젝트는 **프로그래밍 언어(PL)의 원리 학습 및 C 언어 실습**을 위한 공간입니다. 단순한 코드 작성을 넘어 컴파일러, 인터프리터, 메모리 모델, 그리고 AI 코딩 어시스턴트의 활용법 등 컴퓨터 과학의 기초 이론을 실제 코드와 연결하여 학습하는 것을 목적으로 합니다.

## 2. 주요 기술 스택 (Tech Stack)
- **언어:** C (C99/C11 표준 기반)
- **빌드 도구:** GNU Make
- **컴파일러:** GCC (GNU Compiler Collection)
- **디버거:** GDB (GNU Debugger)
- **문서화:** Markdown (학습 노트 관리)

## 3. 빌드 및 실행 방법 (Building and Running)
`makefile`을 통해 프로젝트를 관리하고 있습니다.

- **전체 빌드:**
  ```bash
  make
  ```
  (현재 `say-hello` 실행 파일을 생성하도록 설정되어 있습니다.)

- **개별 컴파일:**
  ```bash
  gcc -c main.c
  gcc -c getname.c
  gcc -o say-hello main.o getname.o
  ```

- **실행:**
  ```bash
  ./say-hello.exe
  ```

## 4. 프로젝트 구조 및 주요 파일 (Key Files)
- **C 소스 파일:**
    - `main.c`: 프로그램의 진입점(Entry Point). 이름 입력 및 환영 메시지 출력 로직 포함.
    - `getname.c`: 사용자로부터 이름을 입력받는 함수 정의.
    - `gugudan.c`: 구구단 출력 실습용 코드 (예상).
    - `bye.c`: 프로그램 종료 또는 작별 인사 관련 코드 (예상).
- **설정 및 문서:**
    - `makefile`: 종속성 관리 및 빌드 자동화 스크립트.
    - `시험공부.md`: 프로그래밍 언어론 강의 요약 및 AI 활용 가이드라인이 포함된 핵심 학습 문서.
    - `아무거나.md`: 임시 메모 또는 추가 학습 내용.

## 5. 개발 및 학습 컨벤션 (Conventions)
- **AI 활용 원칙:** AI를 '정답 자판기'가 아닌 '지능형 튜터'로 활용합니다. 모든 코드에는 이론적 근거를 바탕으로 주석을 달고, 생성된 코드는 반드시 직접 검증(Generate -> Analyze -> Cross-check)합니다.
- **모듈화:** 기능별로 소스 파일을 분리하여 작성하고 `makefile`을 통해 연결하는 구조를 지향합니다.
- **학습 기록:** 새로운 개념을 배우거나 디버깅 과정을 거칠 때 `시험공부.md` 또는 관련 문서를 업데이트하여 지식을 자산화합니다.

## 6. 에이전트를 위한 특이사항
- 이 프로젝트는 사용자의 **학습**에 초점이 맞춰져 있습니다. 코드를 대신 작성해주기보다는 원리를 설명하고, 사용자가 직접 코드를 수정하고 이해할 수 있도록 가이드하는 역할을 우선시합니다.
- `시험공부.md`의 내용을 참고하여 사용자가 배운 용어(고급 언어, 저급 언어, 링커, 로더 등)를 사용하여 소통하십시오.
