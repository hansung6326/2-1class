### 프로그래밍 패러다임

#### 명령형/절차형(imperative/procedural) 패러다임

* 이름있는 데이터를 완전히 명시하고 제어하여 단계적 방법으로 조작하는 것 b = a;

### Cobol(business oriented)

* Common business oriented language

* 1959 ~ 1960, Grace Hopper(최초의 여성 프로그래머, 군인)

* ##### 특징: 낮은 가독성(해석이 어려움), Record 도입, 선언부(declaration) + 실행부(execution)

* 중요성: 비즈니스 관련 시스템 개발에 많이 이용됨, 과거에 잘 나감

---



### Algol(mother language)

* Algorithmic language

* 1950년대 말 ~ 60년대 초

* 범용(general-purpose) 프로그래밍 언어

* Mother language(모태 언어): PL/I, Simula 67, Pascal, C, ...  

* ##### 특징: 블록(block) 구조, 매개변수/ 파라미터/ 인자 전달 //모든 언어가 이 특징을 다 따라 사용함(모태 언어인 이유)

### Pascal(교육용)

* ##### 특징: 프로그래밍 교육용, 블록 구조, 단순함

  * 파일을 다루기 위해서는 사용하기 전에 open

* ##### r-fscanf(infile, "%d", &i); fscanf(infile, "lf", &d); fscanf(infile, "%s", string)

    * 금지: AI를 완벽한 정답지로 맹신하기
  * 과제
    * 권장: AI의 설명을 완벽히 내 것으로 소화해 다시 쓰기
    * 금지: AI의 출력 결과물을 그대로 복사해서 제출하기

### AI워크플로우의 두 갈래: IDE vs Terminal

* ##### IDE형

  * 통합 개발 환경 기반
  * GUI 중심의 시각적 워크플로우
  * GUI 기반 통합 환경
  * Visual Studio Code, Cursor IDE, PyCharm + AI 플러그인
  * 프로젝트 전체 구조 시각화
  * 시각적 디버깅 및 브레이크포인트 제어
  * 코드 자동완성 및 생성
  * 인라인 코드 설명 및 리팩토링 지원
  * IDE형 도구의 실무적 명암(장점)
    * 진입장멱 매우 낮음
    * 코드 컨텍스트 기반의 정밀한 AI 지원
    * 강력한 시각적 디버깅 및 협업 지원
  * IDE형 도구의 실무적 명암(단점)
    * 리소스 점유율 높음, 무거운 환경
    * 대규모 배치 작업에는 비효율적
    * 자동화 및 스크립팅 연동에 불리

* ##### Terminal형

  * CLI 기반
  * 텍스트 명령어 중심의 제어 워크플로우
  * OpenAI CLI, LangChain CLI, Shell GPT
  * 텍스트 기반의 코드 생성 및 즉시 실행
  * 기존 시스템 도구와의 강력한 파이프라인 결합
  * 자동화 스크립트 작성 및 서버 환경 활용 극대화
  * Terminal형 도구의 실무적 명암(장점)
    * 극도로 가볍고 빠른 실행 속도
    * 자동화 및 대규모 배치 처리에 최적화
    * DevOps / MLOps 인프로 환경과 완벽한 친화성
  * Terminal형 도구의 실무적 명암(단점)
    * 높은 학습 곡선
    * IDE 대비 코드 전체 컨텍스트 이해 제한적
    * 시각적 디버깅 도구의 부재

### Cursor AI

* AI 기반 코드 에디터
* VS Code 기반 확장형 개발 환경
* 프로젝트 전체 문맥을 이해하는 LLM 통합

* 단순 자동완성 도구가 아니라 대화형 코드 파트너

* Cursor가 하는 일

  * 코드 생성 및 설명
  * 버그 분석 및 리팩토링 제안
  * 테스트 코드 생성
  * 문서화 자동 생성

* AI가 작성한 코드는 무조건적 '검증 대상'

* 핵심 기능 1: Inline Edit(즉석 코드 생성)

  * Ctrl + K

  * 용도: 코드 창에서 즉석으로 코드 생성 및 수정 요청

* 핵심 기능 2: AL Chat(코드 대화 및 튜터링)

  * Ctrl + L
  * 용도: 코드의 원리 파악 및 질문(1:1 튜터 역할)

* 핵심 기능3: @Symbols (Context Targeting)

  * @Files(특정 파일 내용 참고)
  * @Codebase(프로젝트 전체 구조 및 연관성 파악)
  * @Web(최신 라이브러리/ 기술 인터넷 검색)
  * @Docs(공식 문서 기준 답변 요청)
